Lynx DoH DNS

High-performance DNS-over-HTTPS daemon

What it does

Accepts DNS queries on port 53 (UDP/TCP), resolves them via DNS-over-HTTPS (DoH) to 1.1.1.1, and falls back to plain UDP DNS when the upstream or proxy is unreachable.

Features

DoH client HTTP proxy support Connection pooling (5-15) Adaptive auto-tuning LRU cache (preemptive refresh) Rate limiter (100 qps/client) SIMD name comparison (SSE4.2) Zero-copy DNS parse PID-style auto-tuner Perf monitoring (lock-free)

Quick start

sudo env https_proxy=http://PROXY_IP:PROXY_PORT/ \ ./build/lynx \ --dns_listen_addr=":53" \ --enable_dns_to_https \ --https_upstream="https://1.1.1.1/dns-query" \ --fallback_upstream="GATEWAY_IP:53" \ --log_level=info

Architecture

The server runs on Boost.Asio with a thread pool (4 workers, auto-tuned up to 32). Incoming UDP queries are offloaded to the pool via a lock-free ring buffer. The DoH client maintains a pool of persistent HTTPS connections, auto-tuned between 5 and 15. An AutoTuner PID loop adjusts connections, threads, fan-out parallelism, and cache refresh thresholds every 5 seconds based on real-time latency and error metrics.

Fallback

When DoH fails (proxy down, network issues), queries are forwarded via plain UDP to the configured --fallback_upstream. This ensures DNS resolution continues working even when the internet link through the proxy is unavailable — using the gateway as a bootstrap resolver.

Performance

~112ms uncached sequential • ~43ms cached • 0-copy DNS parsing

Compiled with -O3 -flto -march=native. SIMD-accelerated name comparison, pre-allocated vectors, and connection keep-alive for maximum throughput.

Switch script

sudo lynx-switch start # stop legacy dnss, start lynx sudo lynx-switch stop # stop lynx, restart legacy dnss sudo lynx-switch status # check which is running
