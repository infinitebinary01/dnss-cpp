v0.2.0

2026-06-25

  • Fixed: Proxy path in Connection::open() — missing return true after successful proxy TLS handshake. Execution always fell through to the direct connection block, overwriting the proxy tunnel. The daemon had never actually used the proxy since the fallback was added.
  • Fixed: Prewarm bypassing the cache. prewarmCache() in HttpResolver::init() called resolver->query() directly on the HTTP resolver, bypassing the CachingResolver. Moved to main.cpp and runs through the caching layer so entries populate both L1 turbo and L2 LRU.
  • Fixed: healthCheck() double-decrement bug — removed all mc.connected->store(false) calls from health check. The callers already handle the connected → false transition via exchange(false) with conditional fetch_sub(1). With health check pre-setting the flag, the caller's exchange saw false and skipped the decrement, leaking connectedCount_ by +1 per failure.
  • Changed: Auto-tuner now grows connections based on pool utilization. When utilization > 85% and QPS > 5, adds 1–2 connections immediately (instead of waiting for high latency or errors). Lowers growth thresholds (latency 300→200ms, trend 10→5, error rate 0.03→0.02).
  • Changed: Removed redundant "Resource Usage" section from dashboard GUI — info was already shown in the four summary cards.
  • Performance: Benchmark results with 8 concurrent workers:
    • Cache-hit P95: 2.2ms (well under 100ms target)
    • Mixed workload P95: 67.5ms (under 100ms target)
    • Cache-miss (unique domains): P95 329ms — dominated by PdaNet proxy round-trip latency (~100ms)

v0.1.1

2026-06-20

  • Fixed: Negative active connection count in stats dashboard
    • Connection::exchange() no longer sets connected = false on errors — notifyFailure now exclusively owns the flag via atomic exchange(false) with conditional decrement
    • All three unconditional connectedCount_ decrements (notifyFailure, probeAllIdle, background health check) now use exchange(false) and only decrement when the old value was true
    • Fixed race window by moving inUse = false to after notifyFailure in all callers
